Commands Guide

1. Create a New Virtual Environment

To isolate dependencies for each project or problem, start by creating a new virtual environment:

# Create a new virtual environment in a specified directory (e.g., "env_name")
python3 -m venv env_name

2. Activate the Virtual Environment

Activate the virtual environment for your current shell session:

macOS/Linux:

source env_name/bin/activate

Windows:

.\env_name\Scripts\activate

To deactivate:

deactivate

3. Install Dependencies from requirements.txt

For projects with predefined dependencies, install directly from a requirements.txt file:

pip install -r requirements.txt

4. Create a requirements.txt File for Current Environment

To capture all installed packages for sharing or future use:

pip freeze > requirements.txt

5. Install Common Packages for Machine Learning and Scientific Computing

Here’s a quick setup to install common libraries:

pip install numpy pandas matplotlib scipy scikit-learn tensorflow torch

6. Running Python Scripts

To run a Python script, activate the environment and then use:

python script_name.py

7. Troubleshooting Dependency Conflicts

If you encounter dependency conflicts:

  1. Check for conflicts:
    2. pip check
  2. Upgrade or Downgrade Specific Packages:
    3. pip install package_name==version

8. Problem-Solving with Common Libraries

Using Scipy's dual_annealing for Optimization Problems:

from scipy.optimize import dual_annealing
# Define and run your dual_annealing function here

Using Pandas for Data Manipulation:

import pandas as pd
# Load data
df = pd.read_csv('data.csv')
# Inspect data
print(df.head())
# Manipulate data
df['new_col'] = df['col1'] + df['col2']

Visualizing with Matplotlib:

import matplotlib.pyplot as plt
plt.plot([1, 2, 3], [4, 5, 6])
plt.xlabel('X-axis')
plt.ylabel('Y-axis')
plt.title('Sample Plot')
plt.show()

9. Common Commands for Package Management

List Installed Packages:

pip list

Update a Package:

pip install --upgrade package_name

Uninstall a Package:

pip uninstall package_name

10. Quick Virtual Environment Clean-Up

To quickly clean up and remove the virtual environment if it’s no longer needed:

rm -rf env_name

11. Managing Different Python Versions in Virtual Environments

If you have multiple versions of Python installed, specify the Python version when creating the virtual environment:

python3.9 -m venv env_name

12. Jupyter Notebook Setup in Virtual Environment

To use Jupyter notebooks within a virtual environment:

  1. Install Jupyter:
    2. pip install jupyter
  2. Start Jupyter Notebook:
    3. jupyter notebook

13. Adding Virtual Environment to Jupyter Notebooks

  1. Install ipykernel in the virtual environment:
    2. pip install ipykernel
  2. Add the environment as a new Jupyter kernel:
    3. python -m ipykernel install --user --name=env_name --display-name "Python (env_name)"

14. Creating Reusable Functions for Problem-Solving

Store commonly used functions in a Python file (e.g., utils.py). For example, common matrix or data preprocessing functions.

Import them in your projects:

from utils import function_name

15. Debugging and Inspecting Code

Use Python’s built-in pdb module to add breakpoints:

import pdb; pdb.set_trace()

16. Generating Random Data for Testing

Using Numpy:

import numpy as np
random_data = np.random.randint(0, 100, size=(10, 5))

Using Scikit-Learn for Synthetic Data:

from sklearn.datasets import make_classification
X, y = make_classification(n_samples=100, n_features=5)
